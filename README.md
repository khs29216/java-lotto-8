# 우테코 8기 프리코스 3주차 - 로또

## 기능 구현 목록
### 1. 입력 관련
#### 1-1. 로또 구입 금액 입력
- [X] 사용자로부터 로또 구입 금액을 입력받는다
- [X] 입력받은 로또 구입 금액을 int로 파싱한다
- [X] 파싱된 로또 구입 금액으로 객체를 생성한다
- [X] 입력값 예외 상황을 처리한다
  - [X] 비어있을 경우
  - [X] 공백이 포함된 경우
  - [X] 숫자 이외에 문자가 포함된 경우
  - [X] 0보다 크지 않은 경우
  - [X] 1,000원으로 나누어 떨어지지 않는 경우
- [ ] 예외 상황 시 입력을 다시 받는다

#### 1-2. 당첨 번호 입력
- [X] 사용자로부터 당첨 번호 6개를 입력받는다
- [X] 입력받은 당첨 번호를 리스트로 파싱한다
  - [X] 번호는 쉼표(,)를 기준으로 구분한다
- [X] 파싱된 리스트로 객체를 생성한다
- [X] 입력값 예외 상황을 처리한다
  - [X] 비어있을 경우
  - [X] 공백이 포함된 경우
  - [X] 숫자와 쉼표(,) 이외에 문자가 포함된 경우
  - [X] 입력받은 당첨 번호 중 로또 숫자 범위(1~45)를 벗어난 번호가 있을 경우
  - [X] 입력받은 당첨 번호 중 중복되는 번호가 있을 경우
  - [X] 당첨 번호 6개를 입력하지 않은 경우
- [ ] 예외 상황 시 입력을 다시 받는다

#### 1-3. 보너스 번호 입력
- [X] 사용자로부터 보너스 번호 1개를 입력받는다
- [X] 입력받은 보너스 번호를 int로 파싱한다
- [X] 파싱된 보너스 번호로 객체를 생성한다
- [X] 입력값 예외 상황 처리한다
  - [X] 비어있을 경우
  - [X] 공백이 포함된 경우
  - [X] 숫자 이외에 문자가 포함된 경우
  - [X] 로또 숫자 범위(1~45)를 벗어난 경우
  - [X] 앞서 입력받은 당첨 번호와 중복되는 번호를 입력한 경우
- [ ] 예외 상황 시 입력을 다시 받는다
      
### 2. 사용자 로또 구매
- [X] 사용자가 입력한 로또 구입 금액만큼 로또를 구입한다.
- [X] 사용자가 구매한 금액만큼 로또를 발행한다
  - [X] 로또 1장의 가격은 1,000원이다
  - [X] 각 로또당 랜덤한 번호 6개를 발행한다
    - [X] 로또 번호의 숫자 범위는 1~45이다
      
### 3. 사용자가 구매한 로또 번호와 당첨 번호 비교
- [ ] 각 로또마다 로또 번호와 당첨 번호를 비교한다
- [ ] 1등부터 5등까지 당첨 기준과 비교한다
- [ ] 각 로또마다 몇등인지 저장한다
      
### 4. 출력
#### 4-1. 발행한 로또 출력
- [ ] 발행한 로또의 수량을 출력한다
- [ ] 발행한 로또의 번호를 출력한다
  - [ ] 로또 번호는 오름차순으로 출력한다
     
#### 4-2. 당첨 내역 출력
- [ ] 당첨 내역을 출력한다
  - [ ] 5등부터 순서대로 출력한다

#### 4-3. 수익률 출력
- [ ] 수익률을 출력한다
  - [ ] 수익률은 소수점 둘째자리에서 반올림한다

#### 4-4. 에러 문구 출력
- [ ] 예외 상황 시 에러 문구를 출력한다
  - [ ] 에러 문구는 "[ERROR]"로 시작한다

## 단위 테스트 목록
### 1. 입력 관련
#### 1-1. 파싱 테스트
- [X] (입력값 -> int) 파싱 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 입력값이 비어있을 경우
    - [X] 입력값에 공백이 포함된 경우
    - [X] 입력값에 숫자 이외에 문자가 포함된 경우
- [X] (입력값 -> 리스트) 파싱 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 입력값이 비어있을 경우
    - [X] 입력값에 공백이 포함된 경우
    - [X] 입력값에 숫자, 쉼표(,) 이외에 문자가 포함된 경우
          
#### 1-2. 객체 생성 테스트
- [X] 입력받은 로또 금액 LottoPurchaser 객체로 생성되는지 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 로또 금액이 0보다 크지 않은 경우
    - [X] 로또 금액이 1,000원으로 나누어 떨어지지 않는 경우
- [X] 입력받은 당첨 번호가 Lotto 객체로 생성되는지 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 입력받은 당첨 번호 중 로또 숫자 범위(1~45)를 벗어난 번호가 있을 경우
    - [X] 입력받은 당첨 번호 중 중복되는 번호가 있을 경우
    - [X] 당첨 번호 6개를 입력하지 않은 경우
- [ ] 입력받은 보너스 번호가 LottoWinning 객체로 생성되는지 테스트
  - [ ] 정상적인 입력 테스트
  - [ ] 예외 상황 테스트
    - [ ] 보너스 번호가 로또 숫자 범위(1~45)를 벗어난 경우
    - [ ] 보너스 번호가 당첨 번호와 중복되는 경우
