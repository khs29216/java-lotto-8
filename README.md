# 우테코 8기 프리코스 3주차 - 로또

## 기능 구현 목록
### 1. 입력
#### 1-1. 로또 구입 금액 입력
- [X] 사용자로부터 로또 구입 금액을 입력받는다
- [X] 입력받은 로또 구입 금액을 int로 파싱한다
  - [x] 예외 상황을 처리한다
    - [X] 입력값이 비어있을 경우
    - [X] 입력값에 공백이 포함된 경우
    - [X] 입력값에 숫자 이외에 문자가 포함된 경우
- [X] 파싱된 로또 구입 금액으로 객체를 생성한다
  - [x] 예외 상황을 처리한다
    - [X] 구입 금액이 0보다 크지 않은 경우
    - [X] 구입 금액이 1,000원으로 나누어 떨어지지 않는 경우
- [X] 예외 상황 시 입력을 다시 받는다

#### 1-2. 당첨 번호 입력
- [X] 사용자로부터 당첨 번호 6개를 입력받는다
- [X] 입력받은 당첨 번호를 리스트로 파싱한다
  - [X] 번호는 쉼표(,)를 기준으로 구분한다
  - [x] 예외 상황을 처리한다
    - [X] 입력값이 비어있을 경우
    - [X] 입력값에 공백이 포함된 경우
    - [X] 입력값에 숫자와 쉼표(,) 이외에 문자가 포함된 경우
- [X] 파싱된 리스트로 객체를 생성한다
  - [X] 예외 상황을 처리한다
    - [X] 입력받은 당첨 번호 중 로또 숫자 범위(1~45)를 벗어난 번호가 있을 경우
    - [X] 입력받은 당첨 번호 중 중복되는 번호가 있을 경우
    - [X] 당첨 번호 6개를 입력하지 않은 경우
- [X] 예외 상황 시 입력을 다시 받는다

#### 1-3. 보너스 번호 입력
- [X] 사용자로부터 보너스 번호 1개를 입력받는다
- [X] 입력받은 보너스 번호를 int로 파싱한다
  - [x] 예외 상황을 처리한다
    - [X] 입력값이 비어있을 경우
    - [X] 입력값에 공백이 포함된 경우
    - [X] 입력값에 숫자 이외에 문자가 포함된 경우
- [X] 파싱된 보너스 번호로 객체를 생성한다
  - [X] 예외 상황을 처리한다 
    - [X] 로또 숫자 범위(1~45)를 벗어난 경우
    - [X] 앞서 입력받은 당첨 번호와 중복되는 번호를 입력한 경우
- [X] 예외 상황 시 입력을 다시 받는다
      
### 2. 사용자 로또 구매
- [X] 사용자가 입력한 로또 구입 금액만큼 로또를 구입한다
- [X] 사용자가 구매한 금액만큼 로또를 발행한다
  - [X] 로또 1장의 가격은 1,000원이다
  - [X] 각 로또당 랜덤한 번호 6개를 발행한다
    - [X] 로또 번호의 숫자 범위는 1~45이다
      
### 3. 로또 당첨 등수 체크
- [X] 각 로또마다 로또 번호와 당첨 번호를 비교한다
- [X] 맞은 번호 개수 및 보너스 번호 여부에 따른 등수를 매칭한다
- [X] 각 로또마다 몇등인지 저장한다

### 4. 당첨 통계
- [X] 각 등수마다 몇개 당첨됐는지 체크한다
  - [X] 예외 상황을 처리한다
    - [X] 등수를 체크하지 않은 로또가 있을 경우
- [X] 총 수익률을 계산한다
          
### 5. 출력
#### 5-1. 발행한 로또 출력
- [X] 발행한 로또의 수량을 출력한다
- [X] 발행한 로또의 번호를 출력한다
  - [X] 로또 번호는 오름차순으로 출력한다
     
#### 5-2. 당첨 통계 출력
- [X] 당첨 내역을 출력한다
  - [X] 5등부터 순서대로 출력한다
- [X] 수익률을 출력한다
  - [X] 수익률은 소수점 둘째자리에서 반올림한다

#### 5-3. 에러 문구 출력
- [X] 예외 상황 시 에러 문구를 출력한다
  - [X] 에러 문구는 "[ERROR]"로 시작한다

## 단위 테스트 목록
- UI 로직은 제외하고 테스트를 진행한다

### 1. 입력
#### 1-1. 파싱 테스트
- [X] (입력값 -> int) 파싱 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 입력값이 비어있을 경우
    - [X] 입력값에 공백이 포함된 경우
    - [X] 입력값에 숫자 이외에 문자가 포함된 경우
- [X] (입력값 -> 리스트) 파싱 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 입력값이 비어있을 경우
    - [X] 입력값에 공백이 포함된 경우
    - [X] 입력값에 숫자, 쉼표(,) 이외에 문자가 포함된 경우
          
#### 1-2. 객체 생성 테스트
- [X] 입력받은 로또 금액 LottoPurchaser 객체로 생성되는지 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 로또 금액이 0보다 크지 않은 경우
    - [X] 로또 금액이 1,000원으로 나누어 떨어지지 않는 경우
- [X] 입력받은 당첨 번호가 Lotto 객체로 생성되는지 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 입력받은 당첨 번호 중 로또 숫자 범위(1~45)를 벗어난 번호가 있을 경우
    - [X] 입력받은 당첨 번호 중 중복되는 번호가 있을 경우
    - [X] 당첨 번호 6개를 입력하지 않은 경우
- [X] 입력받은 보너스 번호가 LottoWinningNumber 객체로 생성되는지 테스트
  - [X] 정상적인 입력 테스트
  - [X] 예외 상황 테스트
    - [X] 보너스 번호가 로또 숫자 범위(1~45)를 벗어난 경우
    - [X] 보너스 번호가 당첨 번호와 중복되는 경우

### 2. 사용자 로또 구매 테스트
- [x] 사용자가 입력한 로또 구입 금액만큼 로또가 구입되는지 테스트

### 3. 로또 당첨 등수 체크 테스트
- [x] 로또 번호와 당첨 번호를 비교 후 올바른 등수와 매칭되는지 테스트

### 4. 당첨 통계 테스트
- [ ] 등수별 당첨 횟수가 올바르게 계산되는지 테스트
  - [ ] 정상적인 상황 테스트
  - [ ] 예외 상황 테스트
    - [ ] 등수를 체크하지 않은 로또가 있을 경우
- [ ] 총 수익률이 올바르게 계산되는지 테스트
